Self-Service MetaDL

Each meta-dataset is used to generate few-shot learning tasks via an API from the Meta-dataset repo. For more details, please refer to the provided tutorial. The submission is a dataset that is compatible with the MetaDL data framework. The data should be formatted in the following way:

metadataset
    metatrain
        0.tfrecords
        1.tfrecords
        ...
        dataset_spec.json
    metatest
        85.tfrecords
        86.tfrecords
        ...
        dataset_spec.json
    metadata

Don't forget to include metadata file in the submitted zip file. This file is meant to indicate the CodaLab instance that it should run its ingestion program.

Please consult this repository to have guidelines on how to format the data in the right way. The images should be 64x64x3, if not they will be resized anyway.

Also make sure that the data you use is not following a bi-level specification (e.g. like in Omniglot for Meta-Dataset). This essentially means that we are not considering super-categories to generate episodes.
